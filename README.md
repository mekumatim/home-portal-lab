# home-portal-lab

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面本身不绑定任何特定域名或网站，主要用于实验、演示或作为个人门户的素材集合。

所有页面均为静态 HTML 文件，可直接在浏览器中打开，或托管至任何静态服务器使用。

## 目录说明

```
home-portal-lab/
├── index.html          # 入口页面（可自定义为导航或目录）
├── pages/              # 存放各独立 HTML 页面
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/             # 资源文件（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：核心目录，按需存放不同用途的独立页面。
- `assets/`：存放页面引用的静态资源，可按类型或功能进一步分类。
- `index.html`：可选，作为仓库的默认展示页。

## 页面归档说明

- 每个页面在 `pages/` 下独立存放，文件名应体现页面用途（如 `dashboard.html`、`weather-view.html`）。
- 页面之间无硬编码依赖，可单独部署或浏览。
- 若页面需引用公共资源，建议统一放置在 `assets/` 中，并在页面内使用相对路径引用。

## 维护说明

- 新增页面：在 `pages/` 下创建 HTML 文件，并更新 `index.html` 中的导航（如有）。
- 资源更新：修改 `assets/` 中对应文件后，请确认所有引用该资源的页面能正常工作。
- 删除页面：从 `pages/` 中移除文件，并清理 `index.html` 中的相关链接。
- 提交规范：建议每次提交只涉及一个页面的新增或修改，以便追踪变更。

## 使用方式

1. 克隆或下载本仓库。
2. 直接在浏览器中打开 `pages/` 下的任意 HTML 文件，或使用本地服务器（如 `python -m http.server`）预览。
3. 如需部署至远程服务器，将整个仓库内容上传至静态托管服务即可。
